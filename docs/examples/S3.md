# AWS S3 file shares (optional local example, not distributed)

Optional override for **private, encrypted-at-rest** file ciphertext storage on AWS. The demo uses **local disk by default** ([LocalStorage.md](LocalStorage.md)). Generated S3 files are **gitignored**; only this doc and `.env.s3.example` are versioned.

## Generate local services

```bash
make scaffold-s3-examples
```

Creates (gitignored):

- `demo/symfony8/src/YopassExamples/S3/` — handlers and S3 store
- `demo/symfony8/config/packages/nowo_yopass_s3.local.yaml`
- `demo/symfony8/config/services_s3.local.yaml`
- `demo/symfony8/.env.s3.local` (from `.env.s3.example`)

## Enable in the demo

1. Run `make scaffold-s3-examples` (generates gitignored PHP + YAML).
2. Copy variables from `.env.s3.local` into `demo/symfony8/.env` (include `YOPASS_USE_S3=1`).
3. Install the SDK in the demo:

   ```bash
   cd demo/symfony8 && composer require aws/aws-sdk-php
   ```

4. Configure a **private** S3 bucket:
   - Block Public Access: **on** (all four settings)
   - Default encryption: **SSE-KMS** (set `YOPASS_S3_KMS_KEY_ID`) or SSE-S3
   - IAM policy: `PutObject`, `GetObject`, `HeadBucket` on `arn:...:bucket/name/yopass-shares/*` only

4. Clear cache: `make -C demo/symfony8 update-bundle`

Without `YOPASS_USE_S3=1`, the demo keeps using local disk storage ([LocalStorage.md](LocalStorage.md)).

## Architecture

| Component | Role |
|-----------|------|
| `S3ShareFileHandler` | Enables the file tab; max size from `YOPASS_S3_MAX_FILE_BYTES` |
| `S3EncryptedCiphertextStore` | `PutObject` with `ACL: private` + SSE-KMS or AES256 |
| `S3OffloadingShareRepository` | File shares: DB holds `yopass+s3:…` pointer; blob in S3 |

Ciphertext is already E2E-encrypted in the browser; S3 SSE adds encryption at rest on the bucket side.

## Production

Copy the example classes into your application namespace, adjust IAM/KMS, and set:

```yaml
nowo_yopass:
    file_handler: App\...\S3ShareFileHandler
    database:
        driver: custom
        repository: App\...\S3OffloadingShareRepository
```

Do not commit bucket names, keys, or AWS credentials.
